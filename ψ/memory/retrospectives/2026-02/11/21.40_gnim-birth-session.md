## Honest Feedback

**Friction 1: Subagent symlink permissions.** The /learn skill launched 3 Haiku agents that all failed because they couldn't follow symlinks from ψ/learn/*/origin/ to the ghq directory. This is a known issue in Codespaces — subagents have restricted filesystem access. The workaround (main agent reads directly) worked fine, but it means /learn in its current form is broken in this environment. Future sessions should read ancestor files directly or clone to the working directory instead of using symlinks.

**Friction 2: Timezone mismatch.** The server runs UTC but Ming is GMT+7. The entire awakening was timestamped in UTC first, then had to be corrected. The /awaken skill should ask for timezone in Step 0 context gathering, or detect it from environment. Had to do a separate commit just for timezone fixes.

**Friction 3: No oracle MCP available.** The retrospective template calls for `oracle_learn()` sync, but there's no Oracle MCP configured in this environment. This means learnings aren't being indexed into a searchable database — they only exist as markdown files. Not a blocker, but reduces the power of /trace for future sessions.

## Lessons Learned

1. **Subagent symlink access in Codespaces**: Haiku subagents cannot follow symlinks outside the working directory. Use direct reads from main agent or clone repos into the workspace.
2. **Theme discovery from names**: Sometimes the metaphor is embedded in the name itself. Gnim = Ming reversed = mirror = Oracle principle #3.
3. **Direct reading > agent summaries for philosophy**: Reading philosophy files yourself creates deeper understanding than receiving summarized output from subagents.
